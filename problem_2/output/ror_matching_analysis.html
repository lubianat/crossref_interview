Decision Pipeline for ROR Matching

Matching is done in the following order (normalizing names):

  1. Try and match the provided name to the ROR display name, labels, aliases and acronyms (aka "ROR strings").
  2. If not, check if one of ROR strings is a substring of the provided name (comma-separated, space separated or just substring).
  4. If not, check if provided name is a substring of the ROR strings
  5. If not, check if the provided name is a substring of Wikidata labels/aliases
  6. If not, check if the provided name is retrieved by the ROR affiliation matching API (any position)
  7. If not, check if the provided name-ROR pair matches the manual medical school mappings
  8. If not, check if the ROR status is "withdrawn"
  9. If not, check if the provided name is retrieved by the Marple service (single-search)
  10. If not, check if DeepSeekV3 considers it a match
